# Deepvan 投研 Skill 与组合复盘

这个仓库做两件事。

第一，把 Deepvan 在知乎公开可见的原文、想法、文章和图片表格整理成可复核语料。只收本人公开原文；别人总结、赞同内容、打不开的页面都不进评分。

第二，把“叫兽指数”国内版和国际版拆成动态组合表，再接行情做复盘。这里不把国内和国际硬拼成一个 100%，每个组合自己加总、自己回测。

> 免责声明：本项目只做公开内容整理、结构化抽取和历史复盘，不构成投资建议。

## Skill 抽象

仓库里包含一个可安装的 Codex skill：

```text
skill/deepvan-investment-research/
```

它学习的是投研结构，不是冒充本人发言。调用后应该做这些事：

1. 把问题改写成可交易问题：买、卖、减仓、持有，还是观察。
2. 找主导变量：利率、CapEx、HBM、真实利率、政策、供需、ETF 溢价等。
3. 写传导链：变量怎么传到行业，行业怎么传到盈利或估值。
4. 映射可买资产：美股账户、港股账户、国内基金账户、纯 A 账户分别怎么表达。
5. 给组合动作：加、减、换、对冲、拿现金，不能只说“看好”。
6. 写反证条件：哪个变量变了，这个判断就该撤。

Skill 的输出要保留证据等级：原文全文和 OCR 表是 Tier A；标题、摘要、搜索片段只能做线索。这个原则比较烦，但很必要。

## 怎么复现

安装依赖很少。当前 backtest 脚本只用 Python 标准库。

```bash
python3 tools/deepvan_portfolio_timeline.py
python3 tools/deepvan_portfolio_summary.py
python3 tools/deepvan_portfolio_backtest.py
```

主要输出：

```text
data/portfolio_timeline.holdings.csv
data/portfolio_timeline.events.csv
data/portfolio_backtest.decisions.csv
reports/portfolio_summary_*.md
reports/portfolio_backtest_*.md
```

示例报告在 `examples/`：

- `portfolio_summary_example.md`
- `portfolio_timeline_example.md`
- `portfolio_backtest_example.md`
- `corpus_inventory_example.md`

## 还没做完的地方

- 国内版还要拆 `QDII 主线组合` 的底层持仓。
- 公募基金净值源要接得更稳，最好单独接东方财富 / 天天基金 / akshare 或本地净值缓存。
- 同一篇原文里如果有调仓前后两张表，还要用正文语义判断最终表。
- 文本调仓事件目前是候选层，不能直接当组合权重。
- 评分系统还没有正式把“变量验证”接进来，比如黄金的 TIPS/BEI、创新药的利率敏感度、AI 链的云厂商 CapEx。

现在这版至少先把地基打实了：原文来源可查，组合表能加总，缺口会暴露，回测不会装作什么都知道。
