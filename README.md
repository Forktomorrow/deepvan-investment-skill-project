# Deepvan 投研 Skill 与组合复盘

这个项目从一个很朴素的问题开始：Deepvan 写了那么多投研判断和调仓记录，能不能把它们整理成一套可复核的东西？

不是摘几句金句，也不是让 AI 学他说话。这里真正想做的是三件事：

- 把公开原文、图片表格和 OCR 文本留下来；
- 把“叫兽指数”的国内版、国际版拆成时间序列；
- 用官方净值点校验回测，偏差大的地方回头查 OCR、分红和调仓表。

> 免责声明：这里只做公开资料整理和历史复盘，不构成投资建议。

## 为什么要用官方净值校验

单靠持仓回测容易出错。QDII 有净值滞后，备兑 ETF 有分红，港股 ETF 有溢价，图片 OCR 也会把 `QQQI` 读成奇怪的字符。

所以现在用两条线互相看：

- 官方净值点：来自原文正文，例如“净值从 1.278 回落到 1.263”。
- 持仓重建：来自图片 OCR 表，按权重接行情代理。

目前校验结果还不错。2025-05 到 2026-02 的多数官方点，和持仓重建曲线的偏差在 `±0.6%` 内。比较明显的例外是 2025-12-31，重建值比官方低约 `1.61%`，后面要优先复核 12 月的分红、调仓前后表和年末净值口径。

## 监控怎么接

监控脚本会做四步：搜公开内容、识别新原文、抽调仓事件、发 webhook。

```bash
export ZH_TOKEN="你的知乎开发者 API Key"
export DEEPVAN_NOTIFY_WEBHOOK="你的飞书/QQ/通用 webhook"

python3 tools/deepvan_monitor.py monitor --config config.example.json
```

通知配置在 `config/notify.example.json`，现在支持：

- `feishu`：飞书自定义机器人；
- `qq` / `qqbot`：常见 QQ bot 文本 webhook；
- `generic_json`：通用 JSON webhook。

更详细的配置在 `docs/monitoring.md`。关键词、查询预算、只认本人原文这些口径都可以调。

## Skill 是什么

仓库里带了一个 Codex skill：

```text
skill/deepvan-investment-research/
```

它学的是投研顺序，不是模仿本人。一个比较靠谱的输出应该包含：

- 这个问题到底是买、卖、减仓、持有，还是先观察；
- 主导变量是什么，比如利率、CapEx、HBM、真实利率、ETF 溢价；
- 变量怎么传到行业、盈利、估值和价格；
- 美股账户、港股账户、国内基金账户、纯 A 账户分别怎么表达；
- 什么情况下这个判断该撤。

这部分还会继续迭代。真正有用的 skill，不应该只会说“看好某某方向”，而应该能把判断拆成变量、证据和反证条件。

## 还差什么

- 国内版 2026-06-29 之后的大 QDII 桶还要拆到底层基金。
- 公募基金净值源需要接得更稳，最好单独接东方财富 / 天天基金 / akshare 或本地净值缓存。
- 同一篇原文里如果有调仓前后两张表，还要用正文语义判断哪张是最终表。
- 黄金、创新药、AI 链等主题的变量验证还没完全接进评分系统。

现在这版至少把主干立住了：原文可追，净值有锚，组合能回看，缺口也摊在台面上。
